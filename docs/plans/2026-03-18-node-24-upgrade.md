# Node 24 Upgrade Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Upgrade the declared Node.js version from 20.x to 24.x across all config files and realign TypeScript Node types.

**Architecture:** This is a frontend-only Vite 6 + React 19 + TypeScript 5.8 PWA — Node is used exclusively as a build-time toolchain. All dependency versions (Vite 6, esbuild, Rollup) already support Node 24. Vercel infers the build Node version from `package.json` engines, so no `vercel.json` changes are needed.

**Tech Stack:** Node 24, Vite 6, React 19, TypeScript 5.8, npm, Vercel

**Files:**
- Modify: `.nvmrc`

### Task 4: Install updated dependencies

- Modify: `package-lock.json` (regenerated by npm)

```bash

```

If you see a peer dep warning about `@types/node`, it is non-fatal — Vite and TypeScript both accept any `@types/node` version.

**Step 2: Verify the installed version**

```bash

```

Expected output includes a line like:
```

```
(exact patch version will vary)

**Step 3: Stage the lockfile**

