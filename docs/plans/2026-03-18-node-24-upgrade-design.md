# Node 24 Upgrade Design

**Date:** 2026-03-18
**Status:** Approved

## Context

This app is a frontend-only React 19 + Vite 6 + TypeScript 5.8 PWA deployed on Vercel. Node is used exclusively as the build toolchain (Vite dev server and build). The backend directory exists but contains only empty scaffolding. There is no server-side Node runtime code.

**Current Node version:** `20.x` (set in `.nvmrc` and `package.json` engines)
**Target Node version:** `24.x`

## Approach

Option A — Minimal version bump. Update the three places that declare the Node version and realign the TypeScript types package. No architecture changes required.

## Changes

| File | Change |
|------|--------|
| `.nvmrc` | `20.x` → `24.x` |
| `package.json` engines | `"node": "20.x"` → `"node": "24.x"` |
| `package.json` devDependencies | `@types/node: ^22.14.0` → `@types/node: ^24` |
| `package-lock.json` | Regenerated by `npm install` |

## Rationale

- Vite 6, React 19, TypeScript 5.8, esbuild, and Rollup all support Node 24.
- Vercel reads `engines.node` to select the build runtime — no additional `vercel.json` changes needed.
- Bumping `@types/node` aligns TypeScript's Node type definitions with the actual runtime, eliminating any type mismatches for newer Node APIs.
- No `node_modules` code touches the Node runtime at runtime — only at build time — so compatibility risk is minimal.

## Verification

After applying changes, run `npm run build` to confirm the production build succeeds under Node 24 constraints.
